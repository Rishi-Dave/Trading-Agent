# ADR Index

One line per ADR. ADRs are the authoritative decision record (over auto-memory /
episodic memory). Same-commit rule: an ADR lands with the change it justifies.

- [0001](0001-bootstrap-toolchain.md) — Bootstrap toolchain: uv + single Python package, standalone (no agent_factory), mcp SDK pinned <2, requests-oauthlib, TOML+pydantic caps-no-defaults, pytest-marker wall (blocking day one), launchd, ntfy.sh, system-python secret-guard hook.
- [0002](0002-phase1-oauth-and-server-wiring.md) — Phase 1 OAuth/server wiring: interactive daily OAuth dance (no unattended renewal), in-memory per-run preview→place binding (T2), shared OAuth host is not a prod-path violation, Phase-1 PassthroughGate (labeled, sandbox-only), client-side estimated_cost computation (E*Trade preview has no total-cost field), accountMode=="IRA" is the real retirement signal, refusals returned not raised (FastMCP ToolError wraps/corrupts the payload), fixture scrubbing by value not just by key.
- [0003](0003-phase2-safety-layer.md) — Phase 2 safety layer: gate evaluation order (halts→legality→sizing), UTC calendar day for caps_state, live get_positions() for loss-breaker unrealized P&L (never positions_cache), reset/kill CLI shape (typed confirm + mandatory --operator), ntfy remote triggering amends SPEC §4.3 (TOTP rotating code, not the topic, is the operator proof — revised post-review from a static token that a review found was replayable), ConfiguredSafetyGate's PositionsProvider/StateStore dependency injection, check_preview/check_priced_preview/check_place split, realized_pnl defaults to 0 until Phase 3, sizing gates skip SELL orders (no double-counting exposure), trade_log receipts on every attempted order not just successful ones.
- [0004](0004-phase3-decision-pipeline.md) — Phase 3 decision pipeline: shape spike (multi-analyst, no debate — news/technical/fundamental analysts → deterministic aggregator → trader → advisory risk), LLM/news access via an injected LLMClient/MarketDataSource seam (concrete claude -p adapter built now, wired live only in Phase 4), advisory risk as both a deterministic caps/whitelist/policy mirror and a qualitative LLM step (both non-enforcing, T1), Signal.detail carries source-specific evidence (frozen Decision/Signal dataclasses unchanged), pipeline wall at tests/wall/phase3/ (mirrors the phase1 precedent, informational until phase close), realized_pnl population stays deferred to Phase 4 (execution-coupled, not proposal-coupled).
- [0005](0005-phase4-orchestration.md) — Phase 4 orchestration: runner shape is direct in-process via a shared server.app.build_runtime()/Runtime construction path (SPEC §3.1 import table amended: runner/ may import server/etrade/store/pipeline), OAuth renewal automates idle-timeout recovery (renew_tokens() wired in, best-effort, non-fatal) while the midnight-expiry human dance stays (SPEC §10 resolved), realized_pnl re-deferred a third time (no real fills to compute from), advisory-risk notes now durable via JSONL in the runner's log step, Decision→OrderRequest mapping (HOLD/non-positive/malformed → no order attempt, whitelist deliberately NOT pre-filtered), injectable NotifyFn (never fatal), run wall at tests/wall/phase4/ (mirrors phase1/phase3 precedent, informational until phase close).
- [0006](0006-phase5-observability.md) — Phase 5 observability: daily digest is store-backed (read_caps_state, not the in-memory RunSummary) and fires at the end of run_decision's happy path, status reports written on every exit path best-effort (build_status_report/write_status_report_best_effort, run_id minted up front via server.app.make_run_id), breaker-tripped notification fires at the source (SPEC §3.1 amended: server/ may import notify; ConfiguredSafetyGate takes an injected NotifyFn, fires at most once per UTC day on the fresh-trip branch only, a raising notify never masks gate=loss-breaker), NotifyFn/build_notify moved from runner/decision_run.py to notify/ntfy.py, digest P&L shows live unrealized (shared etrade.models.unrealized_pnl, same calculation the loss-breaker gate uses) labeled distinctly from still-untracked realized_pnl (ADR-0005), no new phase5 wall marker (folds into tests/wall/phase4/ per SPEC §7's own table).
